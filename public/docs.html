API Documentation

Complete API reference for the PDF Generation SaaS platform. This documentation covers all available endpoints, authentication methods, and usage examples.

Base URL

Development: http://localhost:3000
Production: https://pdf.yourdomain.com

Authentication

The API uses two authentication methods:

1. JWT Tokens

Used for user account management endpoints. Include the JWT token in the Authorization header:

HTTP Header 
Authorization: Bearer <your_jwt_token>

2. API Keys

Used for PDF generation endpoints. Include the API key in the X-API-Key header:

HTTP Header 
X-API-Key: <your_api_key>

Response Format

All API responses follow a consistent JSON structure:

Success Response

JSON 
{
  "success": true,
  "message": "Operation successful",
  "data": { ... }
}

Error Response

JSON 
{
  "success": false,
  "message": "Error description"
}

Authentication Endpoints

POST /api/auth/register

Register a new user account.

Request Body

JSON 
{
  "email": "user@example.com",
  "password": "SecurePass123",
  "full_name": "John Doe"
}

Response

JSON 
{
  "success": true,
  "message": "Registration successful! Please check your email.",
  "data": {
    "email": "user@example.com",
    "full_name": "John Doe"
  }
}
Validation
  • Email must be valid and unique
  • Password minimum 8 characters
  • Full name is required
POST /api/auth/login

Login with email and password.

Request Body

JSON 
{
  "email": "user@example.com",
  "password": "SecurePass123"
}

Response

JSON 
{
  "success": true,
  "message": "Login successful",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": {
      "id": 1,
      "email": "user@example.com",
      "full_name": "John Doe"
    }
  }
}

PDF Generation Endpoints

POST /api/generate-pdf

Generate a PDF from HTML content. Requires API Key authentication.

Headers

HTTP Headers 
X-API-Key: your_api_key_here
Content-Type: application/json

Request Body

JSON 
{
  "html": "<html><body><h1>Hello World</h1></body></html>",
  "options": {
    "format": "A4",
    "margin": {
      "top": "20mm",
      "right": "20mm",
      "bottom": "20mm",
      "left": "20mm"
    }
  }
}

Response

Returns PDF file as binary data with appropriate headers.

cURL Example
bash 
curl -X POST http://localhost:3000/api/generate-pdf \
  -H "X-API-Key: your_api_key" \
  -H "Content-Type: application/json" \
  -d '{"html":"<h1>Test</h1>"}' \
  --output result.pdf

Error Codes

Common HTTP status codes and their meanings:

Status Code Meaning Description
200 OK Request successful
400 Bad Request Invalid request parameters
401 Unauthorized Invalid or missing authentication
403 Forbidden Insufficient permissions or quota exceeded
404 Not Found Resource not found
429 Too Many Requests Rate limit exceeded
500 Internal Server Error Server error occurred
